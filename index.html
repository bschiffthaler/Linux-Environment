Environment variables

Bastian Schiffthaler

Most shells - like any other program language - support variables. In bash we can do this


							X=abc123
							# Curly braces are optional but good practice
							echo ${X}
This prints 

							abc123

Environment variables

  • Special variables. Overwriting them (with nonsense) will break your session
  • Store essential information
  • Can be used interactively and in scripts

Important environment variables to know about

  • $PWD - Your current working directory
  • $USER - Your username
  • $HOME - Your home directory
  • $PATH - A list of directories where Linux will look for programs
  • $OMP_NUM_THREADS - Thread parallelism for software supporting OpenMP

Modifying environment variables


						  # Change the number of threads used for OpenMP software
							export OMP_NUM_THREADS=16
							# Add another directory to the binary search PATH
							export PATH=/home/${USER}/bin:${PATH}

This lasts until you log out and only for your current session

Executing programs that are not in your $PATH

  1. Make sure the file is executable with chmod u+x
  2. Start by giving a fully qualified path /path/to/program
  3. We'll learn more about paths in the "Filesystem" lecture

Your working directory $PWD

  • The directory in which your shell currently executes
  • Defaults usually to $HOME after login
  • Check with the pwd builtin
  • Change with the cd builtin
